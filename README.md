# API de E-commerce para Gerenciamento de Produtos 🛒

Este é um projeto prático de desenvolvimento **Backend** que simula as operações essenciais de um e-commerce. A aplicação consiste em uma **REST API** robusta para o cadastro, listagem, atualização e exclusão de produtos, integrando persistência de dados e documentação padronizada.

---

### 🛠️ Tecnologias e Ferramentas Utilizadas

* **Linguagem Principal:** Python 3
* **Framework Web:** Flask
* **Banco de Dados:** SQLAlchemy & SQLite (Banco relacional local)
* **Documentação da API:** Swagger (OpenAPI Specification)
* **Controle de Versão:** Git e GitHub

---

### ⚙️ Funcionalidades Desenvolvidas

* **CRUD Completo de Produtos:** Rotas estruturadas para criar, ler, atualizar e deletar registros no banco de dados.
* **Persistência de Dados Relacional:** Mapeamento de tabelas e gerenciamento de estado dos produtos via ORM (SQLAlchemy).
* **Documentação Interativa:** Arquivo de especificação Swagger para fácil integração, testes de endpoints e visualização da estrutura de dados.

---

### 📁 Estrutura do Repositório

* `app.py`: Arquivo principal contendo a inicialização do Flask, configuração do banco de dados e as rotas da API.
* `instance/ecommerce.db`: Banco de dados relacional SQLite gerado localmente para os testes.
* `1709328020385-attachment.x-yaml`: Arquivo de especificação Swagger contendo toda a documentação dos endpoints.
* `.gitignore`: Arquivo de configuração para garantir que arquivos locais desnecessários ou sensíveis não sejam versionados.
